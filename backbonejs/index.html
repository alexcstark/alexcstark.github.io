BackboneJS and ES6 application

21 May 2016

What I need to understand is concepts, not how to implement.

Thank you Matthew Platts for these videos:

https://www.youtube.com/watch?v=vJwgIth1I_w

Watch all 3 of these videos to begin to conceptualize what Backbone JS is about.

Here's my current understanding:

  • BackboneJS is an ultra-lightweight, un-opinionated, front-end framework.
  • It creates representations of your database within the javascript environment.
    • These representations are attached to items on the DOM so that as you change the JS representations, the DOM is also updated appropriately.
    • After the models have changed, Backbone sends an AJAX request to the server to make the appropriate changes in the database.

During my process of familiarization, I also happened to learn about ES6 arrow functions.

As you watch the videos, you'll notice that Matt needs to create static references to this. He does this by doing var self = this and then referencing self. With ES6, we actually have a fix for this issue!

In the code block below, I've created two methods that showcase how es5 handles this versus es6. 

var _ = require("underscore");  
var people = [  
  {name: "Alex", age: 25},
  {name: "Christine", age: 26}
];

var methods = {  
  es5: function(){
    _.each(people, function(person){
      console.log(this); // window
    });
  },

  es6: function(){
    _.each(people, (person) => {
      console.log(this); // window
    });
  }
}

methods.es5(); // window/global object  
methods.es6(); // methods object

I'll continue to update this with more learnings!
